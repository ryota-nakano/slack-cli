# 変更履歴

## バージョン 2.14.0 (2025-11-28)

### ✨ 新機能

- **メッセージ編集機能**: `/edit <番号>` で自分の投稿したメッセージを編集可能
  - エディタが起動して既存のメッセージを編集
  - 保存して終了すると更新、保存せず終了でキャンセル
  - 自分のメッセージのみ編集可能
  - 例: `/edit 5`

## バージョン 2.13.3 (2025-11-26)

### 🐛 バグ修正

- **`/w`コマンドのURL形式変更**: ブラウザで開く際のURLをアーカイブ形式からクライアント形式に変更
  - 変更前: `https://{workspace}.slack.com/archives/...`
  - 変更後: `https://app.slack.com/client/{teamId}/...`
  - これにより、ブラウザ版Slackアプリで直接開きやすくなりました

## バージョン 2.13.2 (2025-11-26)

### 🐛 バグ修正

- **readline表示バグ修正**: 長文入力時に画面端で折り返すと、入力のたびに改行されて表示が崩れる問題を修正
  - 原因: 画面上の物理行数ではなく論理行数に基づいてカーソル制御を行っていたため
  - 修正: ターミナル幅と文字幅を計算し、物理的な行数に基づいて正確に再描画するように改善
  - `string-width`を使用して全角文字を含む行の長さを正確に計算
  - 削除時に下の行が残る問題を修正（`\x1b[J`を使用）

## バージョン 2.13.0 (2025-11-25)

### ✨ 新機能

- **エディタモードで参照表示**: Ctrl+Eでエディタ起動時に最近のメッセージを参照しながら入力可能
  - 画面を自動的に2分割（上: 参照、下: 入力）
  - 最近10件のメッセージを表示
  - Vim: `-o2`オプションで水平分割
  - Emacs: 自動的に分割表示
  - 参照ファイルは読み取り専用

### 🎨 UI改善

- **`/recent`の表示順序改善**: 日時を先頭に表示
  - 変更前: `[番号] 日時 名前 内容`
  - 変更後: `日時 [番号] 名前 内容`
  - より自然な時系列の視認性

### 💡 使い方の改善

- **エディタの終了方法を明確化**: `:wqa`または`:qa`で全ウィンドウを閉じる
  - `:wqa` → 全ウィンドウ保存して終了（送信）
  - `:qa` → 全ウィンドウ終了（キャンセル）
  - 起動時にヒントを表示

## バージョン 2.12.0 (2025-11-25)

### ✨ 新機能

- **Ctrl+P/Nで履歴を順番に選択**: チャンネル選択画面と`/recent`モードで使用可能

  - より快適なキーボード操作
  - Vimライクな操作感（Ctrl+P/N/J）
  - キーボードのホームポジションから手を離さず操作完了

### 🐛 バグ修正

- **Ctrl+J確定の不具合修正**: Ctrl+Jが改行として認識される問題を修正
  - `key.name === 'enter'`もチェックするように変更（`'return'`だけでなく）
  - Selection modeでは常にEnter/Ctrl+Jで確定
  - デバッグログを追加（`DEBUG_KEYS=1`で有効化）

## バージョン 2.2.4 (2024-11-13)

### 🐛 バグ修正（初期プロンプトの問題）

- **初期プロンプトもredrawInput()で描画**: 手動で`> `を表示していたのが原因
- 最初のプロンプトがscreenCursorLine/previousLineCountに反映されていなかった
- すべての描画をredrawInput()に統一

### 📝 根本原因

初期状態:
```javascript
process.stdout.write(chalk.green('> '));  // 手動で表示
// でも this.screenCursorLine = 0
// でも this.previousLineCount = 1
```

これにより、最初の"> "がクリアされずに残っていた。

修正後:
```javascript
this.redrawInput();  // redrawInputで描画
// screenCursorLine/previousLineCountが正しく設定される
```
